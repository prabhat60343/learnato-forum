## Pre-Deployment

- [ ] Test application locally
- [ ] Commit all changes to Git
- [ ] Push to GitHub repository
- [ ] Prepare environment variables

## Backend Deployment

### Option 1: Render
- [ ] Create Render account
- [ ] Create new Web Service
- [ ] Connect GitHub repository
- [ ] Set root directory to `backend`
- [ ] Set build command: `npm install`
- [ ] Set start command: `npm start`
- [ ] Add environment variables:
  - `PORT=5000`
  - `FRONTEND_URL=https://your-frontend-url`
  - `MONGO_URI=your-mongodb-uri` (optional)
- [ ] Deploy and copy backend URL

### Option 2: Railway
- [ ] Create Railway account
- [ ] Create new project from GitHub
- [ ] Add service (backend folder)
- [ ] Set environment variables
- [ ] Deploy and copy backend URL

### Option 3: Vercel (Backend as Serverless)
- [ ] Create Vercel account
- [ ] Install Vercel CLI: `npm i -g vercel`
- [ ] Run `vercel` in backend directory
- [ ] Set environment variables
- [ ] Deploy

## Frontend Deployment

### Option 1: Vercel (Recommended)
- [ ] Create Vercel account
- [ ] Import GitHub repository
- [ ] Set root directory to `frontend`
- [ ] Set framework preset to Vite
- [ ] Add environment variable:
  - `VITE_API_URL=https://your-backend-url`
- [ ] Deploy

### Option 2: Render (Static Site)
- [ ] Create new Static Site on Render
- [ ] Connect GitHub repository
- [ ] Set root directory to `frontend`
- [ ] Set build command: `npm install && npm run build`
- [ ] Set publish directory: `dist`
- [ ] Add environment variable:
  - `VITE_API_URL=https://your-backend-url`
- [ ] Deploy

### Option 3: Netlify
- [ ] Create Netlify account
- [ ] Import GitHub repository
- [ ] Set base directory: `frontend`
- [ ] Set build command: `npm run build`
- [ ] Set publish directory: `frontend/dist`
- [ ] Add environment variable:
  - `VITE_API_URL=https://your-backend-url`
- [ ] Deploy

## Environment Variables Reference

### Backend
```env
PORT=5000
FRONTEND_URL=https://your-frontend-domain.com
MONGO_URI=mongodb+srv://user:pass@cluster.mongodb.net/dbname
```

### Frontend
```env
VITE_API_URL=https://your-backend-domain.com
```

## Quick Deploy Commands

### Render (using render.yaml)
```bash
# Just push to GitHub, Render will auto-deploy
git add .
git commit -m "Deploy to Render"
git push origin main
```

### Vercel
```bash
# Frontend
cd frontend
vercel

# Backend (if using serverless functions)
cd backend
vercel
```

### Railway
```bash
# Install Railway CLI
npm i -g @railway/cli

# Login
railway login

# Deploy
railway up
```

## Troubleshooting

### CORS Errors
- Update `FRONTEND_URL` in backend environment variables
- Restart backend service

### Socket.io Not Working
- Check if hosting platform supports WebSockets
- Verify Socket.io URL in frontend
- Check browser console for connection errors

### Build Failures
- Check Node.js version (should be 18+)
- Review build logs for errors
- Verify all dependencies are in package.json

### Environment Variables Not Working
- Vite variables must start with `VITE_`
- Rebuild after changing variables
- Restart services after changing backend variables
